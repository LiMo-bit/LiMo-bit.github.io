照搬人家的整好了，在我这里很丑

不过我理想中的 todolist 应该是每一个任务都是可以建立子项的，
类似于整一个遍历逻辑，然后每个父项都是根据子项有没有完成建立的
都可以折叠的那种

前言

博主按照这位大佬的方式整了一个 todolist，但是完全没有我想象中的效果:disappointed:
于是自己开始从头整，todolist 元数据格式还是用的上面的大佬的，有亿点点修改

需求分析

  • 展示内容
    • 必选：任务状态
    • 可选：截止时间
  • todolist 支持 3 级任务
    • 有子项任务的可以进行折叠/展开
    • 父子任务缩进管理
  • 不同状态任务颜色差异
  • 必须在 butterfly 页面下部署，其他页面存在问题
  • hexo+butterfly 的数据管理方式是在_data 目录下新建 yaml 文件，修改不方便
    • 解决方案：Python实现可视化编辑todolist

实现过程

由于博主当前还是属于 web 小白的程度，就按照通用的CLAS(copy learn adapter summary)模型(抄学改结)实现了

COPY

把需求提给 AI，让 AI 帮助实现基本内容

LEARN

AI 提供的页面是使用一个 js 脚本将数据转为 HTML 的
这里的学习目标主要针对 JavaScript 有两点

  1. 将 AI 的 js 中的语法掌握并同样在 pug 中调用 js 实现这部分内容
  2. 将 js 中的语法转为 pug，直接在 pug 中生成对应的 HTML

JavaScript 学习

这里的 TreeView 类主要有下面几个重要部分组成

  1. constructor → 构造函数
    把 todolist 数据和容器节点数据交给 TreeView
  2. render → 渲染函数
    把 todolist 数据转化为 HTML 文本
  3. createTreeView → 创建树节点
    由于我们的数据是一个树形数据，所以这里需要使用迭代的逻辑进行
  4. toggleChildren → 为子节点添加 expend 类
    这里的目的是为了 CSS 处理
  • root 节点
  • 有子任务的节点
  • 没有子任务的节点

通过 toggle 函数切换类 + 点击事件 + CSS 控制实现子节点的展开或者关闭

SUMMARY

document

  • document.getElementById(contanerId)

    getElementById介绍

    通过 id(唯一)获取页面中的元素，这里的 document 指的就是 HTML

  • document.createElement(“div”)

    createElement介绍

    在 HTML 中根据标签名称创建标签，返回值就是标签

  • document.addEventListener(event,func)

    addEventListener介绍

    添加监听时间，当某事件发生时执行对应的函数
    element 也有这个接口

    event.stopPropagation();
    方法会停止当前事件在 DOM 树中的传播，防止事件向上冒泡到父元素。这样可以避免触发父级元素上绑定的相同事件处理程序，实现事件的精确控制。

element

  • appendChild(element)

    appendChild介绍

    添加子元素，参数为 document 创建的元素

  • className

    className介绍

    属性，表示当前元素的类名

  • style

    style介绍

    属性，当前标签的 style

  • querySelector(选择器)

    querySelector介绍

    接口，参数为选择器，如果查找到存在元素，返回该元素，否则返回空

  • childrenContainer.classList.toggle(“expanded”);

    toggle介绍

    classList 当前元素的类列表
    toggle(“expanded”)

    如果有expanded则删除
    如果没有则添加
